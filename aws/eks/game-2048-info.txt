updated to run on aws eks 1.36 and elbv2

To install and run the 2048 game on an existing AWS EKS cluster with 
EC2 managed nodes, follow these steps:

Ensure Prerequisites are met:

kubectl installed and configured to interact with your EKS cluster.

AWS CLI installed and configured with appropriate IAM permissions

tofu is installed

verify the main.tf code, terraform.tf, and variables.tf have the settings you want for your region and cluster setup, 
when verified run:
>tofu init
>tofu plan
>tofu apply

The main.tf will create the aws eks cluster and output a randon eks name when done, this usually takes about 20 minutes

Once the name is output, update your kube config with:

>aws eks update-kubeconfig --region=us-west-2 --name=<name of cluster from output>

Deploy the 2048 Game Application.

>kubectl apply -f 2048-game.yaml

Access the Game:

If using Ingress with AWS Load Balancer Controller:
Wait for the Application Load Balancer (ALB) to provision. Get the Ingress resource and find the ADDRESS (DNS name) of the ALB.

>kubectl get ingress -n game-2048

Open a web browser and navigate to the ADDRESS of the load balancer. No cert is setup in this example, so use http

Open a web browser and navigate to the EXTERNAL-IP to access the game.

to clean up:

run:

>kubectl delete -f game-2048.yaml

confirm the ALB is actually gone with:

>aws elbv2 describe-load-balancers --query "LoadBalancers[?contains(LoadBalancerName, 'game2048')]"

then you can destroy the cluster:

>tofu destroy
